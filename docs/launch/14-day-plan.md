# 14-Day Launch Plan (Show HN First)

## Primary outcome

Get 10+ real users to run the canonical tutorial and provide actionable feedback.

## Day 0 (Launch)

- Publish GitHub release using `docs/launch/release-v0.1.0.md`
- Publish Show HN using `docs/launch/show-hn.md`
- Share short social post from `docs/launch/social-posts.md`
- Monitor for first 4 hours and reply quickly

## Day 1-3 (Onboarding fixes)

- Triage incoming reports daily
- Fix documentation friction same day when possible
- Label issues by track:
  - `open-core`
  - `commercial`
  - `bug`

## Day 4-7 (Feedback release)

- Ship one small follow-up release
- Prioritize:
  - parser robustness
  - error message clarity
  - report readability

## Day 8-14 (Signal synthesis)

- Summarize top 10 feedback items
- Split into:
  - open-core roadmap candidates
  - commercial-track demand signals
- Decide whether to accelerate private commercial build milestones

## Metrics log template

Use this table daily:

| Date | Tutorial completions (proxy) | New issues | Bugs | Open-core requests | Commercial requests | Avg first response time |
|------|-------------------------------|-----------:|-----:|-------------------:|--------------------:|------------------------:|
| YYYY-MM-DD | 0 | 0 | 0 | 0 | 0 | 0h |

## Completion criteria (14 days)

- 10+ credible tutorial completions
- Clear top 3 onboarding friction points identified and addressed
- Clear top 3 commercial demand signals documented
